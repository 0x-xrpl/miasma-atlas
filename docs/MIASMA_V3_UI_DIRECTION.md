# MIASMA V3 UI Direction

This document defines the visual direction for the final MIASMA HTML/UI work.

Do not implement the UI here.

## Design Goals

- premium
- cinematic
- Sui-native
- technical
- clear within 20 seconds
- not generic SaaS
- not a wallet app
- not a DeepBook app
- particle field must have semantic meaning
- motion must support reduced motion

## Section Map

| Section | Visual | Judge takeaway |
|---|---|---|
| Hero Incident | Large incident statement with blocked status and funds moved at 0 | Immediate product truth |
| Agent Field | Proposed action, recipient, amount, and path inputs | The agent intent is visible |
| Mesh Zoom | Focused path zoom on poisoned nodes | The path, not the chat, is the subject |
| Agent Flow Graph | Causal path from draft to scan to gate | The system is pre-execution |
| Micro-drain Escalation | 10-step escalation visual | Small actions can accumulate into risk |
| MIASMA Gate | Threshold decision block | Blocked vs clean is deterministic |
| Wallet Inspection | Approval request state and submit state | Wallet approval is downstream |
| Evidence Capsule | Proof hash, capsule hash, artifact refs | Evidence exists without overclaim |
| Clean Action | Side-by-side clean path state | The same system can allow clean actions |
| What Works Today | Compact truth table | What is live, local, or configured |
| Commercial Path | Small market segments | The product can support real adoption |
| Built through Sui | Ecosystem story panel | The product is Sui-native |
| Final Close | One-line ending with funds moved: 0 | Clear closing thesis |

## Visual Rules

- Keep the incident readable in under 20 seconds.
- Make the micro-drain escalation feel causal, not decorative.
- Make the gate feel like a real policy decision.
- Keep blocked and clean states visually distinct.
- Use semantic color and motion only when it clarifies meaning.
- Honor reduced-motion preferences.
